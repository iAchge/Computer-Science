## Reporting a Vulnerability

If you discover a security vulnerability, please do **not** disclose it publicly.  
Instead, contact me directly:

- **Email:** ghostachge@proton.me  
- **GitHub Security Advisory:**  
  https://github.com/USERNAME/REPOSITORY/security/advisories/new

(Replace `USERNAME` and `REPOSITORY` with your actual GitHub information.)

Please do **not** open a public issue containing vulnerability details.

---

## Responsible Disclosure Process

When you report a vulnerability:

1. We will acknowledge receipt within **72 hours**.
2. We will review and assess the issue as soon as possible.
3. If the vulnerability is confirmed, we will work on a fix or mitigation.
4. We will coordinate with you on a responsible disclosure timeline.
5. Once a fix is released, we will publish a short summary in the release notes or a security advisory.

---

## Scope

This security policy applies to:

- All code and resources contained in this repository  
- Official releases produced for this alarm system project  

Out of scope:

- Forks or modified versions created by others  
- Third-party libraries or dependencies used by the project  
- External hardware or deployment environments (e.g., Raspberry Pi setups outside this repository)
